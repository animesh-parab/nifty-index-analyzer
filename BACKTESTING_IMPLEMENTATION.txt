HOW IT WORKS
================================================================================

WORKFLOW:

1. USER CONFIGURATION
   - Select index (NIFTY/BANKNIFTY)
   - Choose date range (1-60 days)
   - Set interval (5m, 15m, etc.)
   - Define prediction timeframe
   - Set initial capital

2. DATA FETCHING
   - Download historical OHLCV data
   - Calculate technical indicators
   - Filter to market hours
   - Validate data quality

3. BACKTESTING LOOP
   For each candle:
   a. Generate prediction using only past data
   b. Wait for actual outcome (future candle)
   c. Compare prediction vs actual
   d. Simulate trade with P&L
   e. Update capital
   f. Record results

4. METRICS CALCULATION
   - Aggregate all trades
   - Calculate win rate
   - Compute profit factor
   - Find max drawdown
   - Analyze direction accuracy

5. VISUALIZATION
   - Generate equity curve
   - Create distribution charts
   - Display trade history
   - Show detailed metrics

6. EXPORT
   - Save JSON report
   - Export CSV trades
   - Store for analysis

================================================================================
TECHNICAL DETAILS
================================================================================

POSITION SIZING:
- Based on prediction confidence
- Formula: position_size = capital * (confidence / 100) * 0.1
- Maximum 10% of capital per trade
- Higher confidence = larger position

TRADE LOGIC:
- BULLISH → Long position (buy)
- BEARISH → Short position (sell)
- SIDEWAYS → No trade (skip)

P&L CALCULATION:
- Long: P&L = quantity * (exit_price - entry_price)
- Short: P&L = quantity * (entry_price - exit_price)
- Percentage: P&L% = (P&L / position_size) * 100

ACCURACY EVALUATION:
- Bullish correct if actual move > 0
- Bearish correct if actual move < 0
- Sideways correct if |move| < 0.2%

================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: No data fetched
CAUSE: Internet connection, yfinance down, invalid symbol
FIX: Check connection, try again later, verify symbol

ISSUE: Processing too slow
CAUSE: Large dataset, slow computer
FIX: Reduce days, use larger interval, close other apps

ISSUE: Low win rate
CAUSE: Market conditions, model limitations
FIX: Try different periods, adjust timeframe, review logic

ISSUE: Memory error
CAUSE: Too much data
FIX: Reduce days, use larger interval, restart app

ISSUE: UI not showing
CAUSE: Session state issue
FIX: Refresh page, click button again

================================================================================
METRICS EXPLAINED
================================================================================

WIN RATE:
- Percentage of correct predictions
- >60% = Excellent
- 50-60% = Good
- <50% = Needs improvement

PROFIT FACTOR:
- Total wins / Total losses
- >2.0 = Excellent
- 1.5-2.0 = Good
- <1.0 = Losing system

MAX DRAWDOWN:
- Largest peak-to-trough decline
- Lower is better
- Should be <20% of capital

AVERAGE WIN/LOSS:
- Average profit per winning trade
- Average loss per losing trade
- Win should be > Loss

DIRECTION ACCURACY:
- Bullish accuracy: % of correct bullish predictions
- Bearish accuracy: % of correct bearish predictions
- Compare to find strengths
