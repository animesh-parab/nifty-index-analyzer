📋 WHAT IS IT?
──────────────
XGBoost is a machine learning model that learns from your actual trading
outcomes. It replaces the Dual-AI system as the PRIMARY prediction engine
(60% weight) while keeping Groq/Gemini as validation (20% each).

🎯 KEY FEATURES
───────────────
✓ Learns from real market outcomes (not just indicators)
✓ Uses 13 raw indicator values as features
✓ Trains on 300+ logged predictions
✓ Provides direction (UP/DOWN/SIDEWAYS) + confidence (0-100%)
✓ Automatic fallback to Dual-AI if model not trained yet

📦 INSTALLATION
───────────────
1. Install dependencies:
   pip install xgboost scikit-learn

2. Run dashboard:
   streamlit run app.py

3. Let it collect data during market hours (automatic)

⏱️ TIMELINE
───────────
Day 1-3:   Collecting data (need 300+ predictions with outcomes)
Day 3-5:   Ready to train (click retrain button in sidebar)
Day 5+:    Model active, predictions using XGBoost + LLMs

🔄 HOW IT WORKS
───────────────
STEP 1: DATA COLLECTION (Automatic)
  • Every prediction is logged with 13 indicator values
  • After 15 minutes, system checks actual outcome
  • Outcome: UP (+1), DOWN (-1), or SIDEWAYS (0)
  • Saved to: prediction_log.csv

STEP 2: TRAINING (Manual - Click Button)
  • Need 300+ predictions with outcomes
  • Trains XGBoost model on historical data
  • Evaluates accuracy on test set (20% of data)
  • Saves model to: xgb_model.pkl

STEP 3: PREDICTION (Automatic)
  • XGBoost predicts direction + confidence (60% weight)
  • Groq LLM validates (20% weight)
  • Gemini LLM validates (20% weight)
  • Weighted consensus = final prediction

🎯 PREDICTION WEIGHTS
─────────────────────
When XGBoost model exists:
  • XGBoost ML:  60% weight
  • Groq LLM:    20% weight
  • Gemini LLM:  20% weight

When XGBoost NOT trained:
  • Groq LLM:    50% weight
  • Gemini LLM:  50% weight

📈 CONSENSUS LEVELS
───────────────────
STRONG (All agree):
  • XGBoost + Groq + Gemini all predict same direction
  • Confidence: +10% boost
  • Example: All say BULLISH → High confidence

MODERATE (XGB agrees with final):
  • XGBoost agrees with weighted consensus
  • Confidence: Normal
  • Example: XGB=BULLISH, Final=BULLISH

WEAK (XGB disagrees):
  • XGBoost disagrees with weighted consensus
  • Confidence: -15% penalty
  • Example: XGB=BEARISH, Final=BULLISH

🔧 TROUBLESHOOTING
──────────────────
Problem: "Not enough data"
  → Wait for more predictions to be logged
  → Check prediction_log.csv exists
  → Ensure dashboard runs during market hours

Problem: "Model not trained yet"
  → Need 300+ samples with outcomes
  → Check sidebar for progress
  → Wait 15 minutes after predictions for outcomes

Problem: Low accuracy (<50%)
  → Collect more data (aim for 1000+)
  → Retrain with recent data
  → Check if market conditions changed

Problem: "XGBoost module not installed"
  → Run: pip install xgboost scikit-learn
  → Restart dashboard

Problem: Model predictions seem wrong
  → Check if model is stale (retrain)
  → Verify feature values are reasonable
  → Check logs for errors

⚠️ IMPORTANT NOTES
──────────────────
• Model needs 300+ samples minimum (takes 3-5 days)
• Predictions are logged ONLY during market hours
• Outcomes are checked 15 minutes after prediction
• Model file (xgb_model.pkl) must exist for XGBoost to activate
• If model doesn't exist, system uses Dual-AI fallback
• shuffle=False in training (time series data!)

60-65%: Good (profitable with proper risk management)

70%+:   Suspicious (check for data leakage/overfitting)
