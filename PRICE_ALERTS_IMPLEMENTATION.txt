HOW IT WORKS
================================================================================

USER FLOW:
1. User opens dashboard → Sidebar shows "Price Alerts" section
2. User adds alert (e.g., "Above 25,500")
3. Dashboard checks price every 60 seconds
4. When price crosses 25,500:
   - Browser notification pops up
   - Toast notification appears
   - Warning banner shows at top
   - Alert moves to "Recently Triggered"

TECHNICAL FLOW:
1. Alerts stored in price_alerts.json
2. Every refresh (60s), check_alerts() is called
3. Compare current price with all active alerts
4. If condition met, mark as triggered
5. Display notifications using Streamlit's st.toast()
6. Update UI to show triggered alerts

================================================================================
LIMITATIONS
================================================================================

Current Limitations:

1. Dashboard Must Be Open
   - Alerts only work when dashboard is running
   - Browser tab must be open
   - Solution: Keep dashboard open during market hours

2. Browser Notifications Only
   - No SMS or email (yet)
   - No mobile app notifications
   - Solution: Future enhancement (see FUTURE_IMPROVEMENTS.txt)

3. Nifty Only
   - Only works for Nifty 50
   - No Bank Nifty or stocks (yet)
   - Solution: Future enhancement

4. Simple Conditions
   - Only "above" or "below" price
   - No complex conditions (e.g., "above X AND RSI > 70")
   - Solution: Future enhancement

5. No Alert Editing
   - Can't edit existing alerts
   - Must delete and recreate
   - Solution: Future enhancement
